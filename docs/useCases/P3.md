name: User submits the candidate

## Related uses cases
N1, N2.

## Prerequisites
The user is in the `candidateAdd` page.

## Initialicer
The user clicks the button `submit`.

## Steps
1. The data in the page for the `candidate` entity is validated.
2. The `candidate` entity is send to the backend.  
3. The backend responses ok.
4. The `candidateAdd` page is closed, all its data is discarded, and the `main` page is loaded.
