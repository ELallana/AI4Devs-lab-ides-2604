id: N2
name: User submits the candidate but the backend fails
---
## Scope
Frontend.

## Related use cases
P3.

## Prerequisites
The user is in the `candidateAdd` page.

## Initiator
The user clicks the button `submit`.

## Steps
1. The data in the page for the `candidate` entity is validated.
2. The `candidate` entity is sent to the backend.  
3. The backend responses with an error.
4. Show a message on the page in red text indicating the candidate can't be submitted and the problem returned by the backend.
